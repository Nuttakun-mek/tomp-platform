# Contributing

## Working Principles

1. Business rules must not be hidden inside UI components.
2. Significant actions must create timeline events.
3. Published data must not be overwritten without version or event history.
4. Driver UI must stay simple and mobile-first.
5. Every new feature must support TOMP's core goals: reduce time, reduce calls, reduce errors, or improve service quality.

## Branch Naming

- feature/*
- fix/*
- docs/*
- refactor/*
- release/*

## Commit Style

Use clear commit messages:

- `docs: add business rules`
- `feat: add project model`
- `fix: correct assignment status`
- `refactor: separate mission service`
