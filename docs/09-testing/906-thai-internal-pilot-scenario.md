# Thai Internal Pilot Scenario

## วัตถุประสงค์

ทดสอบ flow หลักของ TOMP แบบมีผู้ดูแล ตั้งแต่สร้างโครงการจนถึงตรวจ Timeline โดยใช้ภาษาไทยเป็นหลักและใช้ข้อมูลตัวอย่างที่ทีมปฏิบัติการเข้าใจได้

## เส้นทางทดสอบ

1. เปิดหน้า `ทดสอบ Pilot`
2. เข้า `โครงการ` และตรวจว่ามีโครงการตัวอย่างหรือสร้างโครงการใหม่ได้
3. เข้า project detail แล้วเพิ่มภารกิจ
4. เข้า `Assignment` เพื่อจัดสรร Call Sign รถ คนขับ และช่วงเวลา
5. เปิดลิงก์ QR สำหรับคนขับ
6. คนขับยืนยันชื่อ เบอร์ รถ GPS รูปรถ และรูปป้ายทะเบียน
7. คนขับกดสถานะ `พร้อมเริ่มงาน`, `ถึงจุดรับแล้ว`, `รับผู้โดยสารแล้ว`, `เสร็จสิ้นงาน`
8. ศูนย์ควบคุมเปิด `Mission Control` เพื่อตรวจความพร้อม รายการที่ต้องติดตาม และ Timeline

## ผลลัพธ์ที่คาดหวัง

- ผู้ใช้ไทยสามารถเข้าใจหน้าจอหลักโดยไม่ต้องอธิบายคำอังกฤษเพิ่มเติม
- Driver Card ใช้งานบนมือถือได้และปุ่มสำคัญแตะง่าย
- Timeline แสดงเหตุการณ์โดยไม่มีปุ่มแก้ไขหรือลบ
- ถ้าไม่มี Supabase หรือข้อมูลจริง ระบบยังแสดง `ข้อมูลตัวอย่าง`

## ข้อจำกัดที่ทราบ

- QR image จริงยังไม่ถูกสร้างเป็นภาพ production
- Token security ยังต้องทดสอบกับ secret และ expiry จริงก่อนใช้งานนอกทีม
- Realtime เป็น foundation และ fallback ได้ ยังไม่ใช่ live GPS production
- RBAC และ publish locking ต้องตรวจซ้ำกับผู้ใช้จริงก่อนเปิด Pilot แบบไม่ต้องมีผู้ดูแล
