# Operating Standards

## Planning Standard

Before a plan can be published:

- Requirement must be clear enough for execution.
- Operation days must be defined.
- Sessions must be defined.
- Missions must be defined.
- Required vehicles must be estimated.
- Required drivers must be estimated.
- Call signs must be assigned.
- Coordinator contacts must be defined.
- Google Maps links should be available.
- Conflict check must pass.
- Readiness check must meet project policy.

## Driver Standard

Driver activation includes:

- Confirm identity.
- Confirm phone number.
- Confirm vehicle.
- Take vehicle photo.
- Take plate photo.
- Enable GPS if required.
- Review route sheet.
- Know coordinator contact.
- Confirm readiness.

## Vehicle Standard

Vehicle readiness includes:

- Plate confirmed.
- Vehicle type confirmed.
- Capacity confirmed.
- Vendor confirmed.
- Photo captured when required.
- Out-of-service status checked.

## Coordinator Standard

Coordinator responsibilities:

- Confirm vehicle arrival.
- Confirm passenger boarding.
- Confirm mission completion.
- Report issue or change.
- Contact operation for escalation.
- Capture notes when useful.

## Operation Standard

Operation team focuses on:

1. Monitor exception.
2. Coordinate affected users.
3. Decide recovery.
4. Record action.
5. Preserve knowledge.

## Organizer Standard

Organizer may:

- View allowed project status.
- Submit change request.
- Upload supporting document.
- Contact operation.
- Confirm certain items if granted permission.

Organizer may not:

- Directly change driver.
- Directly change vehicle.
- Directly edit assignment.
