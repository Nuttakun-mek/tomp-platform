# Driver Assignment Delivery Workflow

## Assignment Delivery

1. Operation team publishes assignment.
2. System generates a driver assignment packet.
3. System generates QR/token access.
4. Coordinator sends QR/token to driver.
5. Driver opens assignment in Web Driver or future Driver App.
6. Driver acknowledges the assignment.
7. Driver activates readiness and starts GPS sharing.

## Updates Delivered To Driver

- Time change.
- Pickup change.
- Dropoff change.
- Route change.
- Contact change.
- Cancellation.

## Driver Confirmation

- Received.
- Ready.
- Arrived pickup.
- Passenger onboard.
- Arrived dropoff.
- Completed.

## Mission Control Visibility

- Acknowledged / not acknowledged.
- GPS active / GPS stale.
- Contact needed.
- Route change pending.
- Notification pending.

## No Driver Response

1. Send notification.
2. Record contact event.
3. Escalate to coordinator/control.
4. Replacement recommendation can be added later.
