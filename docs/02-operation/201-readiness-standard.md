# Readiness Standard

## Readiness Levels

```text
Not Started -> Preparing -> Ready -> Verified -> Active
```

## Project Readiness

Measures whether the project is operationally prepared.

Inputs:

- Missions planned
- Assignments created
- Drivers assigned
- Vehicles assigned
- Coordinators assigned
- Contact matrix complete
- Google Maps links complete
- Conflict check passed
- Publish completed

## Driver Readiness

Checklist:

- Driver accessed system
- Name confirmed
- Phone confirmed
- Route sheet viewed
- GPS permission granted
- Vehicle confirmed
- Vehicle photo uploaded
- Plate photo uploaded
- Ready button pressed

## Vehicle Readiness

Checklist:

- Vehicle assigned
- Plate confirmed
- Vehicle type confirmed
- Capacity confirmed
- Vendor confirmed
- Photo uploaded
- Not out of service

## Coordinator Readiness

Checklist:

- Coordinator assigned
- Contact available
- Scope confirmed
- Mission list reviewed
- Key points reviewed

## Readiness Dashboard

Before operations, Mission Control should prioritize readiness over map.

Example:

- Drivers ready: 18/20
- Vehicles ready: 19/20
- Photos complete: 17/20
- GPS active: 15/20
- High risk assignments: 2
