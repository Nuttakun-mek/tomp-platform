# Project Lifecycle

## Lifecycle

```text
Lead -> Requirement -> Planning -> Preparation -> Published -> Operating -> Closing -> Archived
```

## Lead

Potential work not yet confirmed.

Allowed:

- Basic customer data
- Estimated service type
- Draft requirement

Not allowed:

- Driver assignment
- Vehicle finalization
- Publish

## Requirement

Confirmed or semi-confirmed need collection.

Allowed:

- Requirement form
- Contact list
- Service type
- Rough schedule
- Attachments

## Planning

Operation team creates structured plan.

Allowed:

- Operation days
- Sessions
- Missions
- Assignments
- Call signs
- Vehicle types
- Driver planning
- Coordinator planning

## Preparation

Readiness stage before publish/live.

Allowed:

- Driver list
- Vehicle list
- QR generation
- Checklist
- Maps links
- Vehicle photos
- Plate photos

## Published

Plan becomes baseline.

Rules:

- Driver and coordinator can see assigned data.
- Organizer can see permitted scope.
- Direct edits are not allowed.
- Changes must use Change Request.

## Operating

Live operation stage.

Focus:

- Status updates
- GPS
- Timeline
- Notifications
- Incident
- Recovery

## Closing

End-of-project review.

Required:

- Confirm all missions closed.
- Resolve open incidents.
- Capture lessons learned.
- Export reports.

## Archived

Read-only historical record.

Rules:

- No operational edits.
- Knowledge remains searchable.
- Audit remains available.
