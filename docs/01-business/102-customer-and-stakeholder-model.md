# Customer and Stakeholder Model

## Stakeholder Types

### Company Owner / Management

Needs:

- Operational leverage
- Cost control
- Service consistency
- Performance visibility

### Operation Manager

Needs:

- Project overview
- Exception visibility
- Recovery tools
- Decision authority

### Planner

Needs:

- Requirement structure
- Vehicle and driver planning
- Conflict detection
- Publish workflow

### Dispatcher

Needs:

- Live assignment control
- Status board
- Contact access
- Recovery actions

### Coordinator

Needs:

- Assigned scope only
- Confirm arrival, boarding, completion
- Contact driver and operation
- Report incident or change

### Driver

Needs:

- Simple mobile card
- QR access
- Route sheet
- Google Maps links
- Contact numbers
- Issue buttons
- GPS activation

### Organizer

Needs:

- Project visibility if allowed
- Change request
- Contact operation
- Status confidence

### Vendor

Needs:

- Submit vehicle and driver data
- Confirm availability
- Replace vehicle or driver when needed
- Work with operation team

### Passenger

Does not need to use TOMP directly in Version 1. Passenger needs service quality: vehicle arrives on time, driver knows what to do, and service is safe and professional.
