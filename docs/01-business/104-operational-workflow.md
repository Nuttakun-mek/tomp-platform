# Operational Workflow

## End-to-End Workflow

```text
Requirement Intake
-> Project Creation
-> Operation Day Setup
-> Session Setup
-> Mission Planning
-> Resource Planning
-> Assignment
-> Readiness Check
-> Publish
-> Live Operation
-> Change / Recovery
-> Completion
-> Closing Review
-> Knowledge Capture
```

## Planning Workflow

1. Receive requirement.
2. Create project.
3. Create operation days.
4. Create sessions.
5. Create missions.
6. Estimate vehicle and driver needs.
7. Assign call signs.
8. Assign drivers and vehicles.
9. Add coordinator contacts.
10. Add venue and Google Maps links.
11. Run conflict check.
12. Run readiness check.
13. Publish plan.

## Driver Workflow

1. Receive QR or login.
2. Open driver card.
3. Confirm identity and phone.
4. Confirm vehicle and plate.
5. Take vehicle photo.
6. Take plate photo.
7. Enable GPS.
8. Review route sheet.
9. Confirm ready.
10. Operate mission.
11. Update status.
12. Report issue if needed.
13. Complete and release.

## Coordinator Workflow

1. Access assigned scope.
2. Review mission and contact matrix.
3. Confirm vehicle arrival.
4. Confirm passenger boarding.
5. Confirm mission completion.
6. Report issue/change when needed.

## Organizer Workflow

1. View allowed project status.
2. Submit change request if needed.
3. Contact operation if critical.
4. Review progress and confidence.

## Recovery Workflow

1. Detect issue.
2. Create incident if required.
3. Identify impacted mission, assignment, driver, vehicle, call sign, and commitment.
4. Select recovery option.
5. Notify affected parties.
6. Execute recovery.
7. Record outcome.
8. Capture lessons learned.
