# TOMP UI Design System

## Visual Tone

TOMP should feel calm, operational, and trustworthy. The interface is for coordinators, planners, drivers, and operation managers under time pressure, so it prioritizes clarity over decoration.

## Color System

- Operation teal: primary action and active navigation.
- Route blue: navigation links and Google Maps actions.
- Amber: warnings, readiness concerns, pending changes.
- Red: blocked access, failed validation, urgent exceptions.
- Slate: page canvas, borders, body text, secondary controls.

## Typography

Use clear sans-serif UI typography. Headings should be compact and readable; avoid hero-scale type inside operational panels.

## Spacing And Cards

Cards use 6-8px radius, subtle borders, and light shadows. Do not nest cards inside cards. Page sections use clear vertical rhythm and responsive grids.

## Button Hierarchy

Primary actions use operation teal. Secondary actions use white background with slate border. Destructive or blocked states use red text/borders.

## Status Colors

Ready: teal. Warning: amber. Critical/blocked: red. Informational/live: blue.

## Dark Mode

Dark mode is deferred. Current pilot uses a light operations workspace optimized for desktop control rooms and mobile driver cards.

## Responsive Behavior

Operation pages are desktop-first but must remain readable on tablets. Driver pages are mobile-first.

## Accessibility Basics

Use semantic buttons and links, visible focus states, sufficient contrast, and text labels for operational actions.
