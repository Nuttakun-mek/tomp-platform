# Thai Copy Guideline

## Core Terms

- Project: โครงการ
- Mission: ภารกิจ
- Assignment: งานที่จัดสรร, or Assignment when the operational object name matters
- Driver: คนขับ
- Vehicle: รถ
- Mission Control: ศูนย์ควบคุมปฏิบัติการ
- Timeline: ลำดับเหตุการณ์ Timeline
- Readiness: ความพร้อม
- Exception: รายการที่ต้องติดตาม
- Demo/fallback data: ข้อมูลตัวอย่าง

## Tone

Use formal but easy-to-understand Thai. The UI should feel direct, calm, and operational. Avoid long explanations inside operational screens.

## Allowed English Terms

TOMP, Call Sign, QR, GPS, Google Maps, and Mission Control only when paired with Thai context.

## Microcopy Rules

- Prefer action verbs: เปิด, ตรวจ, สร้าง, ยืนยัน, แชร์, ติดตาม.
- Show missing data as “ยังไม่ระบุ”.
- Show risk as “ต้องติดตาม” rather than vague warning.
- Do not claim production-ready while the system is still Internal Pilot.
